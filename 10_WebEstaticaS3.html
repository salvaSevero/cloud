    Web estática con S3.¶

    • Vamos a generar una web estática que se almacenará en un S3.

    ¿Qué es una web estática?

    Son páginas que son totalmente idénticas para todos los usuarios. Solo se modifica si el desarrollador edita.

    Crear el S3 (Bucket).¶

    • Elige un nombre único en todo AWS para tu bucket.
    • Permite el tráfico público.
    • Activa el versionado.

    Establece la página estática.¶

    • Accede a tu bucket y dentro a la pestaña propiedades.

    • Activa la opción de "Static website hosting" o "Alojar un sitio web estático".

      💥 La página inicial se llamará index.html.

    • Accede a la web desde el enlace una vez activada la opción.

    Sube los ficheros de la página web estática a tu S3.¶

    Desde la página principal de nuestro bucket en la pestaña Objetos podemos arrastrar la web.

    La web estática la podéis descargar del Aules.

    Generamos Políticas de Seguridad.¶

    • Dentro del Bucket accedemos a Permisos ➡️ Política de Bucket.

    • Vamos a usar el Generador de Políticas para que nos ayude a confeccionar la medida de seguridad.

      💥 Es importante que tengamos en cuenta el ARN de nuestro Bucket.

    ARN.

    El ARN es una cadena de texto que identifica de forma única dentro de AWS nuestro objeto, en este caso un S3.

    Generador de Políticas.

    Las políticas creadas con el generador no se aplican automáticamente, son simplemente una muestra que podremos copiar.

    Vamos a generar una política que permita a los clientes ver nuestra página pero que no puedan borrar/modificar/crear recursos en el S3.

    • Principal: Indica los usuarios que podrán acceder a la página.

      💥 Con * permitimos cualquier usuario.

    • Actions: Qué acciones permitimos hacer a los usuarios.

      💥 Con GetObject sólo les permitimos leer.

    • ARN: Indica dentro del bucket a qué recursos se aplica esta regla.

      💥 Con arn:aws:s3:::mi-cafe-web-cloud/* estamos haciendo referencia a todos los obetos del bucket.

    • Conditions: Podemos agregar condiciones, por ejemplo limitar el acceso por IP.

    Recuerda pinchar en add statement para agregar la regla.

    • Una vez generada la política debemos copiarla y pegarla en la pestaña de políticas.

    Recuerda guardar los cambios para que se apliquen.
